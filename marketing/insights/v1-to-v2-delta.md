# v1 → v2 Changes

## Messaging Improvements
- Moved the opening toward the repo-grounded problem statement that retrieval paths are often opaque, based on the interpretation note that inspectability is the strongest distinctive angle.
- Repositioned CrispyBrain more explicitly as a lab, learning system, and workflow clarity tool because the repo repeatedly says it is early, build-in-public, and not a finished platform.

## Clarity Fixes
- Made the simple `.txt` ingest → question → inspect loop more prominent because the ingest guide gives that as the clearest observable behavior.
- Replaced broader "AI memory" phrasing with more concrete descriptions of storing notes, retrieving them, and showing sources/trust metadata, based on the likely confusion note around vague memory language.

## Misconceptions Addressed
- Stated earlier that the project is not production-ready or turnkey because the interpretation notes flagged a likely risk that "local-first" and "self-hosted" could be misread as product maturity.
- Clarified that the current default path is strongest in the local `alpha` setup and that plain text is the safest ingest path today, to avoid over-reading document support or setup maturity.

## Stronger Angles
- Brought grounded retrieval, source visibility, and trust visibility closer to the top because those are the clearest repo-supported differentiators.
- Kept the honesty section but sharpened it because the repo consistently emphasizes current limitations and that tone is likely to resonate more than polished product language.
